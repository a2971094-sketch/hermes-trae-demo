## 🔧 任务: 角色权限与用户管理

### 分支
feat/role-permissions

### 需求描述
在已有的用户认证模块基础上，实现角色权限中间件和管理员接口

功能要求:
1. 角色权限中间件
   - get_current_user 依赖: 从 Authorization header 解析 JWT，返回 User 对象
   - admin_required 依赖: 验证当前用户 role 是否为 "admin"，否则返回 403
   - 把这两个依赖放到 app/deps.py

2. 管理员用户管理接口
   - GET /api/admin/users — 返回所有用户列表（仅管理员）
   - DELETE /api/admin/users/{id} — 删除指定用户（仅管理员）
   - 注册时通过查询参数 ?role=admin 支持创建管理员账号（仅首次使用有效，之后忽略）

3. 用户资料接口
   - GET /api/users/me — 返回当前登录用户的信息（需认证）

4. 更新或新建测试覆盖新接口

技术要求:
- 复用现有的 User 模型和数据库
- 新增测试覆盖率 > 80%
- 代码符合 Black + ruff 风格
